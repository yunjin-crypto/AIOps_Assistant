\# AI运维助手测试知识库



\## 第一章 Kubernetes Pod故障处理



\### CrashLoopBackOff



当Pod状态出现CrashLoopBackOff时，表示容器启动后立即退出，并进入持续重启状态。



常见原因：



1\. 应用启动失败

2\. 配置文件错误

3\. 环境变量缺失

4\. 数据库连接失败

5\. 端口占用

6\. 镜像版本错误



排查命令：



```bash

kubectl describe pod <pod-name>

```



查看日志：



```bash

kubectl logs <pod-name>

```



查看历史日志：



```bash

kubectl logs --previous <pod-name>

```



标准处理流程：



步骤1：查看Pod事件



步骤2：查看容器日志



步骤3：检查ConfigMap



步骤4：检查Secret



步骤5：检查镜像版本



步骤6：检查依赖服务



\---



\### ImagePullBackOff



ImagePullBackOff表示镜像拉取失败。



常见原因：



1\. 镜像名称错误

2\. 镜像仓库不可达

3\. 镜像不存在

4\. Harbor认证失败

5\. DockerHub限流



排查命令：



```bash

kubectl describe pod

```



检查：



Events字段。



\---



\### Node NotReady



Node NotReady表示节点失去正常工作能力。



检查命令：



```bash

kubectl get nodes

```



查看节点详情：



```bash

kubectl describe node <node-name>

```



常见原因：



1\. kubelet异常

2\. 容器运行时异常

3\. 网络故障

4\. 磁盘满

5\. 内存不足



处理步骤：



重启kubelet：



```bash

systemctl restart kubelet

```



查看状态：



```bash

systemctl status kubelet

```

\## 第二章 Redis运维手册



\### Redis主从同步失败



现象：



slave状态持续显示：



```text

master\_link\_status:down

```



排查命令：



```bash

redis-cli info replication

```



重点字段：



master\_link\_status



slave\_repl\_offset



master\_repl\_offset



常见原因：



1\. 网络中断

2\. 防火墙阻断

3\. replication backlog不足

4\. 主节点重启

5\. Redis版本不兼容



处理流程：



第一步：



检查网络连通性



```bash

ping master-ip

```



第二步：



检查端口



```bash

telnet master-ip 6379

```



第三步：



重新同步



```bash

slaveof no one

slaveof master-ip 6379

```

\### Redis内存占满



查看内存：



```bash

redis-cli info memory

```



重点指标：



used\_memory



used\_memory\_human



maxmemory



常见原因：



1\. key数量过多

2\. 大Key

3\. 缓存未淘汰



检查大Key：



```bash

redis-cli --bigkeys

```



推荐淘汰策略：



```text

allkeys-lru

```



不推荐：



```text

noeviction

```

\## 第三章 Nginx运维指南



\### 502 Bad Gateway



502错误表示上游服务异常。



检查：



```bash

tail -f /var/log/nginx/error.log

```



常见原因：



1\. 后端服务未启动

2\. 后端超时

3\. 网络不通

4\. upstream配置错误



处理步骤：



检查后端端口：



```bash

netstat -lntp

```



检查配置：



```bash

nginx -t

```



重新加载：



```bash

nginx -s reload

```



\---



\### 504 Gateway Timeout



504表示网关超时。



优化参数：



```nginx

proxy\_connect\_timeout 60;

proxy\_send\_timeout 60;

proxy\_read\_timeout 60;

```



常见原因：



1\. SQL执行过慢

2\. Redis阻塞

3\. 网络延迟
